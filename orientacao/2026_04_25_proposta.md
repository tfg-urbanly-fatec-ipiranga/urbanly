# Proposta de Uso de IA Leve na Busca Semântica — Urbanly

## 1. Contexto

Durante a evolução do Urbanly, foi levantada a questão de que o conceito de **“vibe”** ainda estava pouco claro:  
como o sistema poderia interpretar a vibe desejada pelo usuário a partir de uma busca?

A partir dessa problematização, buscamos uma forma de transformar essa ideia subjetiva em algo tratável pelo sistema.

Foi nesse contexto que identificamos o uso de **embeddings**, uma técnica que permite representar significado de forma computacional.

---

## 2. O que são Embeddings

Um **embedding** é uma representação numérica (vetorial) de um dado do mundo real, como um texto.

No contexto do Urbanly:

- o termo de busca do usuário → vira um vetor numérico  
- os dados dos estabelecimentos → também viram vetores  

Esses vetores representam o **significado do conteúdo**, ou seja, uma aproximação matemática da “vibe”.

---

## 3. Relação com o conceito de “Vibe”

A “vibe” de um lugar é algo subjetivo, difícil de representar diretamente com campos fixos.

Com embeddings, conseguimos lidar com isso da seguinte forma:

- a vibe não precisa ser explicitamente definida  
- ela é **implicitamente capturada pelo significado do texto**

Exemplo:

Busca do usuário:
> "lugar tranquilo para conversar"

O sistema entende essa intenção e encontra locais como:

- cafés silenciosos  
- ambientes com baixa movimentação  
- espaços confortáveis  

Ou seja, a “vibe” é inferida a partir do significado da busca.

---

## 4. Como isso ajuda na busca

Com embeddings, o sistema deixa de comparar apenas palavras exatas e passa a comparar **significados**.

Isso permite:

- interpretar linguagem natural  
- lidar com sinônimos  
- aproximar a busca da intenção real do usuário  

---

## 5. Como os embeddings são utilizados no sistema

O funcionamento ocorre em duas etapas principais:

### 5.1 Geração offline (dados do sistema)

Os embeddings dos estabelecimentos são gerados previamente e armazenados no banco de dados.

Utilizando:

- nome do estabelecimento  
- descrição  
- tags  

Essa etapa ocorre:

- quando um estabelecimento é criado  
- quando seus dados são alterados  
- quando suas tags são atualizadas  

---

### 5.2 Geração online (busca do usuário)

Quando o usuário realiza uma busca:

1. o termo digitado é convertido em embedding em tempo real  
2. esse vetor é comparado com os vetores dos estabelecimentos  
3. o sistema identifica os mais semelhantes  
4. os resultados são ordenados por relevância  

---

## 6. Comparação entre embeddings

Como embeddings são números, eles podem ser comparados matematicamente:

- vetores próximos → significados semelhantes  
- vetores distantes → significados diferentes  

Isso permite identificar quais estabelecimentos possuem maior compatibilidade com a busca.

---

## 7. Uso de IA no processo

Para gerar os embeddings, utilizamos um **modelo de IA pré-treinado**.

Importante:

- não é necessário treinar o modelo  
- o modelo apenas transforma texto em vetor  
- pode ser utilizado via API ou localmente  

Isso caracteriza o uso de **IA leve**, pois:

- não envolve treinamento de modelos  
- possui baixo custo computacional  
- mantém a arquitetura simples  

---

## 8. Benefícios para o Urbanly

- torna o conceito de “vibe” operacionalizável  
- melhora a relevância da busca  
- permite busca por linguagem natural  
- reduz dependência de palavras exatas  
- aproxima o sistema de soluções modernas  

---

## 9. Resumo simplificado

- a “vibe” deixa de ser algo abstrato  
- passa a ser representada pelo significado do texto  
- esse significado é convertido em números (embeddings)  
- o sistema compara esses números  
- e retorna os lugares mais compatíveis  

---

## 10. Conclusão

O uso de embeddings permite resolver diretamente o problema levantado sobre a definição de “vibe”, transformando um conceito subjetivo em uma representação computacional.

Dessa forma, o Urbanly evolui de um sistema baseado em palavras-chave para um sistema capaz de interpretar intenções e sugerir experiências alinhadas ao perfil do usuário.
